# Distillation of SWE Agent Trajectories into Small Models

This repository contains the materials for the research project on distilling trajectories from large software engineering (SWE) agents into smaller language models.

The project is motivated by the growing use of agentic frameworks for code generation and debugging, such as OpenHands and SWE-Gym. Our goal is to study how to transfer useful behavior from large-model trajectories into smaller models through supervised fine-tuning, trajectory filtering, and trajectory chunking.

## Project goal

We investigate how to train compact models on agent trajectories generated by larger models and how the choice of trajectory processing affects the final quality.

## What is included

This repository is intended to collect:

* analysis of the Nemotron-SWE-v1 dataset,
* preprocessing and splitting into train/test sets,
* trajectory chunking experiments,
* trajectory filtering experiments,
* loss curves and qualitative comparison of training setups,
* the workshop paper draft in ACM format.

## Datasets

The main dataset used in the experiments is:

* **NVIDIA Nemotron-SWE-v1**

## Methods

We explored several ways to adapt long SWE-agent trajectories for small models:

### 1. Chunking trajectories

Long trajectories were split into smaller chunks to make them more suitable for training compact models with limited context length.

### 2. Filtering trajectories

We also tried two filtering variants to reduce noisy or low-value parts of the data and keep more informative samples for fine-tuning.

### 3. Masked supervised fine-tuning

The training setup follows an instruction-style supervised fine-tuning scheme where only selected parts of the trajectory are used as prediction targets, while the rest is masked.

## Models and training

The experiments were run with a small language model Qwen3-4B and a fine-tuning workflow based on QLoRA via Unsloth.

## Reproducibility

To reproduce the experiments, you will generally need:

1. Python environment with notebook and training dependencies.
2. Access to the selected SWE trajectory dataset.
3. Fine-tuning setup for small LLMs.
4. The preprocessing pipeline for chunking or filtering trajectories.
5. Evaluation scripts for validation on the held-out test split.

## Paper highlights

The paper summarizes:

* the dataset analysis,
* the chunking and filtering strategies,
* the training behavior of small models on SWE trajectories,
* the practical difficulties of evaluating trajectory quality using exact-match metrics,
* the motivation to move toward sandbox-based agent evaluation.

## Notes

* The current work is exploratory and focuses on distillation strategies for agent trajectories.
* Exact-match trajectory metrics are informative but often too strict for agentic behavior.
* Future work should emphasize sandbox evaluation, where an agent is tested on whether it actually solves a task inside an execution environment.
